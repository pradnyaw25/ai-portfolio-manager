Architecture & Retrospective

How the fund is built

An explainable, LLM-run paper hedge fund — in public. A LangGraph-orchestrated daily cycle sits on a hardened LLM gateway, with idempotent stores as the system of record, Qdrant as long-term memory, evals in CI, and every decision traced, cited, and scored.

The fund analyzes markets, debates, trades a simulated $1M book, and justifies itself — every run. Deterministic guardrails (position sizing, turnover, sector limits, stop-loss/take-profit) sit outside the model; the LLM must argue its case and respond to the bear thesis before anything executes. The differentiator isn't returns — it's radical transparency.

The daily cycle

Twenty guarded LangGraph nodes. Every LLM claim is grounded before it's journaled or tweeted; every deterministic guardrail runs before execution.

Mark-to-market→ Research context→ Memory retrieval→ Tool-calling research→ Debate → PM decide→ Grounding check→ Risk review→ Rebalance→ Human approval→ Execute→ Journal→ Report + tweet→ Memory ingest
Conditional routing: empty decision or all-rejected → skip execution, straight to journaling
Resume: a killed run re-enters by run_id; idempotent stores dedupe; an already-sent tweet isn't re-posted
Failure: any node error → finalize with diagnostics + exported status

  • Config consolidation; watchlist & sectors in YAML

✓ Phase 1

Orchestration & observability

  • LangGraph is the runner; conditional routing + diagnostics
  • Human-in-the-loop approval gate
  • Langfuse tracing, per-run cost, durable history
  • SQLite progress store → crash-safe resume

  • Golden-scenario decision evals gating CI

  • Brier score + calibration curves on the dashboard

  • Grounded weekly investor letter

Key engineering decisions

The calls worth remembering — where a constraint forced a choice, and why we went the way we did.

DecisionWhy
Idempotency-based resume over LangGraph's native SqliteSaverThe graph state carries non-serializable live handles (engine, clients). Persisting progress + reusing the run_id leans on the P0-3 idempotent stores for duplicate-free re-execution — correct, without a large risky state refactor.
Trades keyed by the run, not (run_id, symbol, action)A single run can legitimately hold two same-symbol/action trades (a PM buy + a rebalance top-up). A per-row key would silently collapse them and lose shares.
Chunking eval with a deterministic hashing embedder + in-memory QdrantRuns in CI with no API key, yet does real vector search. Cosine over TF vectors rewards term concentration — the actual reason chunking helps — so the 0.15 → 1.00 hit@1 win is honest, not rigged.
Research augments, doesn't replace the deterministic contextKeeps the base context reliable/auditable while still demonstrating the tool-calling stack.
Real EDGAR 8-K earnings exhibits over a paid transcript feedOfficial, free, and reuses the existing SEC client; full call transcripts aren't available free.
Deterministic risk exits win over the LLMStop-loss/take-profit SELLs are generated from the book and supersede any LLM trade on that symbol — the guardrail layer is boring on purpose.
Grounding gate before publish for the investor letterA letter whose numbers aren't supported by the week's computed facts is blocked, never published.
Paper trading — simulated capital, not investment advice. Source & full roadmap in the repository; the fund is also queryable from any MCP client (Claude Desktop / Code) via the read-only server.
